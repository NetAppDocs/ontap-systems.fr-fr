Sur certains systèmes, les ports compatibles RoCE sont automatiquement configurés pour une utilisation de stockage lorsqu'une carte PCIe ou un module d'E/S compatible RoCE est installé dans un emplacement de contrôleur pris en charge. Cependant, vous devez suivre cette procédure pour vérifier que les ports sont correctement configurés pour une utilisation de stockage.

La configuration des ports RoCE non dédiés pour le stockage est une procédure non perturbatrice. Il n'est pas nécessaire de redémarrer les contrôleurs, sauf si l'un ou les deux sont en mode maintenance. Cette procédure suppose qu'aucun contrôleur n'est en mode maintenance.

Si vous devez à l'avenir modifier l'utilisation des ports pour le stockage en vue de leur utilisation réseau, utilisez la commande `storage port modify -node _node_name_ -port _port_name_ -mode network`.

.Étapes
. Vérifiez que vous avez installé toutes les cartes PCIe ou modules d'E/S compatibles RoCE supplémentaires dans chaque contrôleur.
. Connectez-vous au cluster à l'aide d'une connexion SSH ou du port série console.
. Vérifiez que les ports non dédiés de la paire haute disponibilité sont configurés pour le stockage en saisissant la commande `storage port show`.
+
Le résultat de la commande varie en fonction de votre version d'ONTAP :

+
** **ONTAP 9.8 ou version ultérieure :** Les ports non dédiés configurés pour le stockage affichent  `storage` dans la  `Mode` colonne. Les ports non configurés pour le stockage affichent  `network` dans la  `Mode` colonne.
** **ONTAP 9.7 :** Les ports non dédiés configurés pour le stockage affichent  `false` dans la colonne  `Is Dedicated?` et  `enabled` dans la colonne  `State`. Les ports non configurés pour le stockage affichent  `false` dans la colonne  `Is Dedicated?` et  `disabled` dans la colonne  `State`.


. Si les ports non dédiés sont configurés pour le stockage, cette procédure est terminée. Sinon, suivez les étapes suivantes pour configurer les ports.
. Configurez les ports non dédiés pour le stockage sur l'un des contrôleurs. Vous devez répéter la commande correspondante pour chaque port que vous configurez.
+
[cols="1,3"]
|===
| Si votre paire haute disponibilité est en cours d'exécution... | Utilisez cette commande... 


 a| 
ONTAP 9.8 ou version ultérieure
 a| 
`storage port modify -node _node_name_ -port _port_name_ -mode storage`



 a| 
ONTAP 9.7
 a| 
`storage port enable -node _node_name_ -port _port_name_`

|===
. Répétez l'étape précédente pour la deuxième manette.
. Vérifiez que les ports non dédiés des deux contrôleurs sont configurés pour le stockage en saisissant la commande `storage port show`.
+
Le résultat de la commande varie en fonction de votre version d'ONTAP :

+
** **ONTAP 9.8 ou version ultérieure :** Les ports non dédiés configurés pour l'utilisation du stockage affichent  `storage` dans la  `Mode` colonne.
** **ONTAP 9.7 :** Les ports non dédiés configurés pour l'utilisation du stockage affichent  `false` dans la  `Is Dedicated?` colonne et  `enabled` dans la  `State` colonne.
